Movie Webpage Project

Context

Me and partner for this project were supposed to be making a made up movie webpage about whatever we wanted using pretty much the most significant elements we learned about HTML and CSS. So, the movie we decided to make was a thriller that is actually based off of an arc, or season if you will in the anime "One Piece", but we ended up naming it "Lost Bark" and making it our own.

Content

For the HTML elements we had to include the following:

  • heading(s)
  • paragraph(s)
  • div(s) and pan(s)
  • google font(s) that haven't been used in class
  • input(s) and submit

For the CSS though, we had to include:

  • selectors:element and class and id
  • box model: padding and border and margin

We also had to add creative commons (or original) images, we had to include at least 2, and they had to be local. Obviously we had to have partner evaluation as well, so that was it for what we had to include. Now lets move on to the reflection.

Reflection

Challenges

The first challenge me and partner had was with trying to find a place to put the `id` attritbute because we had everything but that, so the id attribute didn't work no matter where we put it. Then, we remembered how the id attrbute only is for one specific word, so we put it for a word that we only used once and it worked.

Another challenge that we had was that no matter how many times I clicked on our link to the website, it couldn't load in. So I tried to make a different yellkey, it didn't work. Thought I used the wrong link, and it still didn't work. But, when I told my partner to change the yellkey instead since he created the pickcode, it finally worked.

Takeaways

I learned more about how to use an `id` attribute through having trouble trying to only use it for a word that was said in our website once, and about it's significance in CSS that I completely forgot about in our last lessons. This was how the `id` attribute is one of the more important elements that can override CSS.

It also made me want to use slack more because with slack, you and your partner, or just classmates in general, can chat with each other about updates/ catching each other up.

Also to make a plan.md before your project so you can know who is doing what since that's what everybody did for their project, and it helped me and my partner with us just not being random and doing a poor job.
